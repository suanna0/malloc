Implicit List

An implicit list is a way to organize free memory blocks in a heap. The list of blocks is implicit—it exists simply because blocks are stored next to each other in memory.

We know that a block has a header that contains its size and allocation status. When looking for a free block, the allocator scans through the heap linearly to find a suitable block.

mm.c is the file you’ll be working on for this lab. When you first open the github repo, mm.c will contain a very simple, albeit not very fast nor efficient, implementation of malloc, the implicit list. Before we go over any changes to make, we will talk about what each of the functions in mm.c do.
