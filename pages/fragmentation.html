Fragmentation

Fragmentation occurs when unused memory is unable to be allocated–this could be caused by alignment requirements, the block’s minimum size, headers/footers taking up space, and much more. There are two forms of fragmentation: internal fragmentation and external fragmentation.

Internal fragmentation is when the size of the allocated block is larger than the payload it holds. This may happen if the implementation has a minimum size on allocated blocks that is greater than the size of the payload.

External fragmentation happens when the overall-sum of free memory blocks across the heap is enough to allocate a block, but no single, contiguous, free block is large enough to handle the request.
