Segregated List

Changing my implemention from an explicit list to a segregated lists was the ultimate test to see if my coalesce_block function was actually clean. My original implementation was full of nasty if-statements, but adding a second free list forced me to clean up my code.

For this assignment, you are given 128 bytes of global variables, so use up all of them for your size classes. The size classes should be organized by powers of two. I played with instrumentation to determine my classes, using a counter to variable see which sizes are used the most frequently and updating the size classes accordingly.

These changes helped, but my implementation was still unable to meet the desired throughput metrics. This led me to modify my find_fit function. I set a variable, max_attempts so that a suitable block could be returned faster, even though it may not be the best fit. Although this change hindered utilization, the throughput improved drastically.
