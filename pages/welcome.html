Welcome!

This site serves as a guide to help demystify some key concepts. You’ll be writing a dynamic memory allocator library which includes malloc, free,  realloc, and calloc, functions you’ve seen in 15-122 and lab0 from earlier in the semester.

In Checkpoint, you will implement an explicit list then move to segregated lists, which will improve the library’s throughput. In Final, you will create optimizations to reduce the minimum block size which will improve the library’s utilization. It’s important to note that this site primarily provides high-level conceptual guidance. However, it is not unusual for the the small nuances in debugging to take up a lot of time–the purpose of the assignment it seeks to to help improve students’ ability to debug complex issues.

General Advice:
Avoid trying to complete everything in one sitting. It’s hard to stay focused for extended periods, so take regular breaks—this means starting earlier to give yourself time to step away and come back with fresh eyes.

When implementing major changes to your library’s functionality, review all functions in the codebase and make a list of the changes you’ll need to make. The more specific and detailed this list, the better.

In terms of time investment, the scoring heuristic increases non-linearly. Don’t let that discourage you. Most of the effort goes into debugging, while the points primarily come from making performance optimizations.
