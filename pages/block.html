Block

Let’s first go over what a block looks like. A block has a one-word header (8 bytes) at the beginning, a payload, and extra padding to fit alignment requirements. Typically, blocks are aligned by 8 or 16 bytes to work well with the computer’s hardware. We’ll be using 16 byte alignment.

In the block’s header, we store information about the size and allocation status. For now, we can imagine that the header stores the size of the block.

When blocks are fixed to the 16-byte alignment requirement, the three lowest-order address bits will always be 0. We can use this space to store a bit that marks the block’s allocation status. A mask can help us get the allocation status of our block.
